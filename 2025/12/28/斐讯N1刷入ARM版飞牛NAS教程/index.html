斐讯N1刷入ARM版飞牛NAS教程

2025-12-28 技术教程 NAS, 刷机教程, 斐讯N1, 飞牛OS 0 评论

前言

最近在折腾低功耗 NAS 方案时，发现斐讯 N1 这类老设备依然有不错的可玩性。本文依据 B 站 UP 主 进恩哥 发布的《斐讯N1刷入arm版飞牛nas教程以及资料》一文进行整理，对刷机流程、关键步骤及相关资源进行系统化记录，便于后续查阅和复现。

原文地址：
https://www.bilibili.com/opus/1150776795526594567

一、刷机前准备

在开始之前，请确认已准备好以下内容：

  1. 斐讯 N1 设备一台
  2. 8GB 及以上 U 盘一个（建议空盘）
  3. 飞牛 NAS ARM 版系统镜像（.img 文件）
  4. U 盘写入工具（如 Rufus）
  5. 一台可进行操作的电脑

建议在刷机前备份原系统相关资料，避免不可逆风险。

二、制作启动 U 盘

  1. 将下载好的系统镜像解压，得到 .img 文件
  2. 打开 Rufus
  3. 选择目标 U 盘
  4. 加载镜像文件
  5. 点击“开始”，等待写入完成

写入完成后，该 U 盘即可作为斐讯 N1 的启动盘使用。

三、U 盘启动斐讯 N1

  1. 将制作好的 U 盘插入 斐讯 N1 的 USB 接口

    • 注意优先插靠外侧的 USB 口

  2. 给设备通电并启动

  3. 等待系统加载（首次启动可能需要 1～2 分钟）

启动成功后，屏幕或日志中会显示设备当前获取到的 IP 地址。

四、写入系统到 eMMC

通过 SSH 或终端方式连接到 N1 后，进入系统环境。

执行以下命令检查安装脚本是否存在：

root@debian# ls

如果看到 install-to-emmc.sh 脚本，则执行：

./install-to-emmc.sh

该脚本会自动将飞牛 NAS 系统写入 N1 的 eMMC 内部存储。
等待脚本执行完成，终端会提示：

Complete copy OS to eMMC

此时：

  1. 关闭设备
  2. 拔掉 U 盘
  3. 重新上电启动

五、刷机完成后的访问方式

设备重启后：

  • 通过浏览器访问 N1 的 IP 地址
  • 即可进入飞牛 NAS 的 Web 管理界面

说明事项：

  • 当前版本多为内测版本
  • 应用商店可能暂不可用
  • Docker 功能通常可以正常使用
  • 数据盘可通过外接 U 盘或硬盘扩展

六、相关资源整理

原作者提供的相关资料与下载地址如下（以实际可访问内容为准）：

  • 资源主页：
    https://jinenge.us.kg

  • 123 云盘资料合集：
    https://www.123pan.com/s/ZNN8Vv-vngc3

七、总结

斐讯 N1 虽然硬件性能有限，但作为一台低功耗、低成本的轻量级 NAS 设备仍具备一定价值。通过刷入 ARM 版飞牛 NAS，可以满足基础文件存储、Docker 容器等使用需求。

刷机过程涉及启动介质和 eMMC 写入，具有一定风险，建议在充分理解流程的前提下操作。后续如果有更多玩法或优化方案，也可以在此基础上持续折腾。
